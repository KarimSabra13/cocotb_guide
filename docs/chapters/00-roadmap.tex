\label{ch:roadmap}

\chapterplaceholder{This repository is intentionally in skeleton mode. The book structure, visual tooling, and asset pipeline are ready before drafting the real content.}

\section*{Planned Chapters}
\begin{itemize}
  \item Verification context and where \cocotb{} fits in a modern DV stack.
  \item The cocotb execution model, scheduler behavior, and coroutine patterns.
  \item Drivers, monitors, scoreboards, and reference-model architecture.
  \item Clocking, resets, synchronization, sampling, and timing discipline.
  \item Randomization, coverage thinking, and stimulus design.
  \item Debug workflows, waveform reasoning, and root-cause analysis.
  \item Regression structure, reproducibility, automation, and CI integration.
  \item End-to-end case studies with reusable verification patterns.
\end{itemize}

\section*{Figure Strategy}
\begin{itemize}
  \item Use TikZ, CircuitikZ, PGFPlots, and TikZ Timing for LaTeX-native publication-quality figures.
  \item Use the Python generator for repeatable SVG assets driven by YAML specs.
  \item Keep source specs in \texttt{docs/figures/specs} and generated assets in \texttt{docs/figures/generated}.
\end{itemize}
